# Security Policy

## Supported Versions

We actively support and patch security vulnerabilities in the following versions of Verfix:

| Version | Supported |
|---------|-----------|
| >= 0.1.x| ✅ Yes    |
| < 0.1.0 | ❌ No     |

## Reporting a Vulnerability

We take the security of our runtime and users seriously. If you discover a security vulnerability, please report it to us responsibly. 

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please report security issues by emailing:

📧 **security@verfix.dev**

Please include the following information in your report:
- **Component**: Website, documentation, CLI, or backend runtime.
- **Description**: A clear description of the vulnerability.
- **Exploit / PoC**: Steps to reproduce the vulnerability (including code/payloads if possible).
- **Impact**: What could an attacker achieve with this vulnerability?

### Our Commitment

We will:
1. Acknowledge receipt of your report within 48 hours.
2. Investigate the issue and determine appropriate fix steps.
3. Keep you updated on our progress toward a patch.
4. Provide a fix/patch in a timely manner (typically within 14 days, depending on severity).
5. Credit you for the discovery in our release notes (unless you prefer to remain anonymous).
